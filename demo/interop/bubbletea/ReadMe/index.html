    Bubbletea Interop Demo¶

    A terminal UI grocery picker built with Bubbletea, demonstrating Go/Clojure interoperability via Gloat.

    Running¶

    make build
./picker

    Use j/k (or arrow keys) to navigate, space/enter to toggle items, d when done, q to quit.

    What This Demonstrates¶

    Bubbletea is a Go TUI framework built around the Elm architecture: a model, an update function, and a view function. It requires implementing the tea.Model interface, which has three methods: Init(), Update(), and View().

    The Core Difficulty¶

    Glojure (the Clojure-to-Go compiler underlying Gloat) does not support deftype or reify, so there is no way to implement a Go interface directly from Clojure code. This rules out the obvious approach of writing the entire Bubbletea application in Clojure.

    The Hybrid Solution¶

    The solution splits responsibilities across two files:

    picker.clj — Pure functional Clojure containing all application logic:

    • init-model — returns the initial state map
    • update-model [model key-str] — pure function: takes old state and a key string, returns new state (or nil to signal quit)
    • format-result [model] — returns the formatted result string when done, nil otherwise
    • view-model [model] — returns the full TUI display string

    The Clojure state is an opaque persistent map. Go never inspects its contents directly.

    shim.go — A minimal Go shim that:

    • Implements tea.Model with a struct { state interface{} } field
    • Delegates all logic to Clojure via glj.Var("picker.core", "fn-name").Invoke(...)
    • Detects the "done" signal by calling format-result after each update
    • Prints the formatted result after the TUI exits

    Calling Clojure from Go¶

    After the Glojure runtime is initialized and picker.core is required, functions are resolved by name at startup:

    updateModelFn = glj.Var("picker.core", "update-model")
result := updateModelFn.Invoke(m.state, msg.String())

    glj.Var returns a lang.IFn — Glojure's callable interface. Clojure persistent maps, keywords, strings, and nil cross the boundary as plain Go interface{} values, making the interop straightforward.

    Signaling State Transitions¶

    Rather than inspecting the Clojure map from Go (which would require keyword lookups via the Glojure API), the Go shim calls format-result to check whether the user pressed d. format-result returns nil when not done and a formatted string when done — a clean, idiomatic boundary: Go sees only Go types.

    Build Process¶

    shim.go is a plain Go file; it hardcodes the demo's module path (gloat-demo/picker) so it can be built with go build directly. The Makefile is a thin convenience wrapper:

    1. gloat --module=gloat-demo/picker picker.clj -o build/ compiles Clojure to a Go module directory, including a generated main.go
    2. cp shim.go build/main.go replaces Gloat's main.go with the shim
    3. go mod tidy adds Bubbletea, then go build produces the binary

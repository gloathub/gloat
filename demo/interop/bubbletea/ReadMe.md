Bubbletea Interop Demo
======================

A terminal UI grocery picker built with
[Bubbletea](https://github.com/charmbracelet/bubbletea), demonstrating
Go/Clojure interoperability via Gloat.

## Running

```bash
make build
./picker
```

Use `j`/`k` (or arrow keys) to navigate, `space`/`enter` to toggle items,
`d` when done, `q` to quit.

## What This Demonstrates

[Bubbletea](https://github.com/charmbracelet/bubbletea) is a Go TUI framework
built around the Elm architecture: a model, an update function, and a view
function.
It requires implementing the `tea.Model` interface, which has three methods:
`Init()`, `Update()`, and `View()`.

### The Core Difficulty

Glojure (the Clojure-to-Go compiler underlying Gloat) does not support
`deftype` or `reify`, so there is no way to implement a Go interface directly
from Clojure code.
This rules out the obvious approach of writing the entire Bubbletea application
in Clojure.

### The Hybrid Solution

The solution splits responsibilities across two files:

**`picker.clj`** ŌĆö Pure functional Clojure containing all application logic:

- `init-model` ŌĆö returns the initial state map
- `update-model [model key-str]` ŌĆö pure function: takes old state and a key
  string, returns new state (or `nil` to signal quit)
- `format-result [model]` ŌĆö returns the formatted result string when done,
  `nil` otherwise
- `view-model [model]` ŌĆö returns the full TUI display string

The Clojure state is an opaque persistent map.
Go never inspects its contents directly.

**`shim.go`** ŌĆö A minimal Go shim that:

- Implements `tea.Model` with a `struct { state interface{} }` field
- Delegates all logic to Clojure via `glj.Var("picker.core", "fn-name").Invoke(...)`
- Detects the "done" signal by calling `format-result` after each update
- Prints the formatted result after the TUI exits

### Calling Clojure from Go

After the Glojure runtime is initialized and `picker.core` is required,
functions are resolved by name at startup:

```go
updateModelFn = glj.Var("picker.core", "update-model")
result := updateModelFn.Invoke(m.state, msg.String())
```

`glj.Var` returns a `lang.IFn` ŌĆö Glojure's callable interface.
Clojure persistent maps, keywords, strings, and `nil` cross the boundary
as plain Go `interface{}` values, making the interop straightforward.

### Signaling State Transitions

Rather than inspecting the Clojure map from Go (which would require keyword
lookups via the Glojure API), the Go shim calls `format-result` to check
whether the user pressed `d`.
`format-result` returns `nil` when not done and a formatted string when done ŌĆö
a clean, idiomatic boundary: Go sees only Go types.

### Build Process

Because `main.go` is a custom file (not generated by Gloat), the Makefile
uses a two-step process:

1. `gloat --module=... picker.clj -o build/` ŌĆö compiles Clojure to a Go
   module directory, including a generated `main.go`
2. `perl` replaces the generated `main.go` with `shim.go`, substituting
   the `GO-MODULE` placeholder with the actual module name
3. `go mod tidy` adds Bubbletea, then `go build` produces the binary
